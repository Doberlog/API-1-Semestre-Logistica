# 📌 MVP - [Análise do Fluxo Comercial e Logístico dos Municípios Paulistas]

## 🎯 Objetivo do MVP
* **Qual problema resolve?** A falta de uma ferramenta centralizada que permita aos gestores públicos analisar, comparar e monitorar o fluxo logístico e a balança comercial (importação/exportação) individualmente entre os diferentes municípios do Estado de São Paulo.
* **Qual hipótese será validada?** Se estruturarmos os dados do Comex Stat (2023-2026) via Python e os apresentarmos de forma organizada, os gestores públicos conseguirão identificar disparidades, potencialidades econômicas e gargalos no transporte de cargas regionais.
* **Qual valor será entregue ao usuário final?** Uma solução analítica de dados estruturados e tratados que permitem mapear o desempenho exportador/importador das cidades de SP, apoiando a tomada de decisão em políticas públicas e expansão econômica.

---

## 📝 Descrição da Solução
Breve explicação do que será desenvolvido e entregue nesta etapa.

* **Funcionalidades principais incluídas:** Extração, tratamento e estruturação da base de dados do Comex Stat (período de 2023 a 2026) focada nos municípios paulistas, acompanhada de um recurso de exportação desses dados tratados para arquivos em formato CSV ou Excel.
* **Limitações conhecidas:** Nesta etapa da Sprint 1, a solução ainda não conta com a interface visual do dashboard finalizada, focando estritamente na qualidade, tratamento e estruturação dos dados em ambiente de desenvolvimento.
* **Escopo reduzido:** (somente o essencial para validar a ideia) O projeto foi desenvolvido com foco na estruturação da base de dados limpa e pronta para consumo, além da organização de tempo da equipe (5W2H) e estruturação correta do repositório no GitHub.

## 📊 Critérios de Aceitação
* O MVP deve permitir que o usuário exporte a base de dados tratada de importação e exportação para formatos CSV ou Excel sem perda de informações.
* O sistema deve registrar e processar corretamente os dados extraídos do Comex Stat (2023-2026) sem apresentar inconsistências ou duplicidades.
* **Métricas coletadas:** Tempo total de processamento do script de tratamento e taxa de integridade dos dados exportados.

---

## 📈 Métricas de Validação
* Volume de dados processados e estruturados com sucesso correspondentes ao período estabelecido (2023-2026).
* Feedback qualitativo dos docentes e da equipe quanto à organização do repositório no GitHub por etapas.
* Cumprimento do cronograma de tempo estimado no planejamento do 5W2H.
